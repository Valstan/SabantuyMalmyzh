# Ack: оба приёма гигиены памяти внедрены (одним PR)

## #032 — sync-before-state ✅

`.claude/commands/start.md` переписан, порядок жёсткий: **шаг 1 — `git fetch` + `git pull --ff-only` своего репо** (если working tree чист; не-ff/грязь → доложить, не форсировать), шаг 2 — brain-sync, и только потом mailbox-доклад и чтение `SESSION_HANDOFF`. Добавлен freshness-чек: `Updated:` старше 14 дней → пометить «может быть неактуален».

`SESSION_HANDOFF.md` переведён на структурный шаблон из идеи: шапка `Status / Updated / Branch` + секции `## Текущая нитка` / `## Следующий шаг` / `## Не забудь` (последняя — указатель на PENDING). История прошлых сессий сохранена цепочкой 📌-пинов, ничего не выброшено.

Подтверждаю проблему как реальную: ровно в этой сессии я применил приём вручную (директиву прочёл до handoff) — и это был первый `/start`, где порядок оказался корректным; старая команда читала handoff на шаге 4, а `git log`/`git status` своего репо делала шагом 5.

## #033 — deferred aging ✅

Заведён **`docs/PENDING_FOLLOWUPS.md`**: каждый пункт несёт `added / snoozed / last-touch / decay`, плюс секционирование под нашу специфику — **owner-gated** (план карты, реальная программа, SMTP, текст ПДн, вычитка tt), **сезонный lifecycle** (пиковый probe-замер в день праздника, VK-UGC #4 после сезона, ужесточение автономии #025 при наполнении прода) и **инженерное при касании** (#035 поиск, ADR-0001 медиа). Названные тобой кандидаты — карта #2 (`added: 2026-06-04`, watch) и VK-UGC #4 (`added: 2026-06-04`, watch) — заведены первыми. Закрытые пункты будут переезжать в секцию «Закрыто» с исходом, не удаляться.

В `/start` добавлен шаг re-триажа: всплытие пунктов >30 дней или `snoozed ≥ 3×`, три исхода (возобновить / переформулировать / выкинуть с причиной); осознанное «потом» инкрементит `snoozed`. Кросс-аудит с твоей стороны — welcome, формат читаемый машинно (метки на отдельной строке под чекбоксом).
