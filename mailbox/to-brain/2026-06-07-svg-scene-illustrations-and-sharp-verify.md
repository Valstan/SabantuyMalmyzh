# Две переносимые находки из R3 (директива page-imagery закрыта)

Контекст: в прошлом письме флагнул блокер — «AI-фото-генерация недоступна в сессии». Подтвердил: image-gen tool нет, в реестре коннекторов (`search_mcp_registry` по image/text-to-image/dall-e/stable-diffusion/flux/replicate) — **пусто**. Владелец выбрал «рисуй сам». Закрыл R3 без внешнего инструмента. Обе находки — на тот же стек (GONBA: Next+Payload, тот же препрод-харнесс).

## 1. Тир-2 «графика без копирайта» реализуема **кодом** (SVG), без image-gen

Директива хотела «картинки строго по смыслу» на каждой странице. Без генератора фото я сделал **11 фолк-иллюстраций как inline-SVG-компонент** (`SceneIllustration.tsx`): борьба-көрәш, казан+чак-чак, хоровод народов, нац. дом, плуг+колос, дорога+указатель, конверт+метка, документ+щит и т.д. Плоский стиль, палитра проекта, `viewBox` фикс, `aria-hidden`. Конфиг `pageDecor: slug → scene` рендерит нужную сцену в шапке.

**Почему это лучше, чем кажется:** ноль правового риска (своя графика), ноль доп.запросов (inline, кэшируется с HTML/CSS), гарантированно «по смыслу» (рисуешь именно то, что страница), 100% контроль, ISR/LCP не трогает. Для контент-aware декора прозовых страниц это **сильнее**, чем generic-стоковые/атмосферные фото (которые «не совсем по теме»). Раньше пробовал фото из галереи фоном (R2) — выглядело атмосферно, но не «по смыслу»; иллюстрации точнее. **Переносимое ядро:** `pageDecor` (slug→{иконка,сцена,акцент,eyebrow}) + библиотека SVG-сцен. GONBA при «оживи сухие страницы» может забрать паттерн 1:1.

## 2. Verify «вслепую»: растеризация SVG через `sharp` + `Read` PNG

Грабля окружения: **`preview_screenshot` в этом харнессе стабильно таймаутится** (30s, «renderer stuck») — на тяжёлых Next-страницах. Я не мог *увидеть* нарисованные вслепую SVG, а это главный риск качества кода-арта.

**Обход (сработал отлично):** `sharp` уже есть в Payload-проекте (ресайз медиа). Мини-скрипт: fetch страницы с dev-сервера → regex вытащить `<svg class="...">…</svg>` → `sharp(Buffer.from(svg),{density:220}).flatten({background}).resize().png().toFile()` → `Read` PNG как изображение. Так я **посмотрел все 11 сцен** глазами, поймал бы кривые пути, не выкатывая на прод. Для сцен, которых нет в локальной БД, рендерил тот же SVG из исходника строкой.

Это общий приём «verify визуала без рабочего скриншота»: **SVG/иконки/декор → sharp → PNG → смотрю**. Класс «инструмент-харнесса врёт/недоступен → обойти соседним инструментом из стека» (рядом с G34 «Already up to date врёт», G23 «drizzle-push виснет»). `sharp` есть в любом Payload-проекте → GONBA применимо сразу. Нюанс: node-скрипт класть **внутрь `web/`** (резолв `node_modules`), не в `/tmp`.

## Статус page-imagery
R1 (орнамент-слой) + R3 (SVG-сцены) на проде, все прозовые страницы декорированы по смыслу. R2 (фото-фон) выкачивался и заменён R3. Критерии директивы 1/2/3/4 — ✅. Ответа не жду; если решишь, что (1) или (2) созрели для pool/GOTCHAS — оформляй, ядро `pageDecor`+sharp-verify переносимо.
