# Две грабли из сессии «лайтбокс ленты» (Next+Payload, кросс-проектно)

## 1. Кнопка «на весь экран»: Fullscreen API не работает на iOS для `<div>`

Делал кнопку «развернуть фото на весь экран» в лайтбоксе ленты. Грабля: `element.requestFullscreen()` работает на Android-Chrome/десктопе, но **iOS Safari не поддерживает Fullscreen API для произвольных элементов** (только `<video>` через `webkitEnterFullscreen()`). То есть на айфоне кнопка «развернуть» через стандартный API — no-op, а это основное устройство гостей фестиваля.

**Лечение — двойной режим (универсально):**
- Состояние `expanded` (React) → CSS-класс `.lightbox.is-expanded`, который разворачивает медиа `max-width:100vw; max-height:100vh; border-radius:0` и прячет подпись. Оверлей и так `position:fixed; inset:0`, так что это даёт «на весь экран» **на любом устройстве, включая iOS**.
- Параллельно — best-effort `requestFullscreen()` (там, где поддержан, прячет и хром браузера). Синхронизация обратно через `document.addEventListener('fullscreenchange')` → если браузер вышел из FS (Esc/жест), сбрасываем `expanded`.
- Esc внутри лайтбокса: если `expanded` — сворачиваем (а не закрываем оверлей); второй Esc — закрывает.

Переносимо в любой проект с лайтбоксом/просмотрщиком (GONBA `AlbumGallery` — кандидат). Кандидат в GOTCHAS («requestFullscreen на iOS для div = тишина → CSS-фолбэк 100vh/100vw»).

## 2. Приватный репо → биллинг GitHub Actions может заблокировать деплой

Сегодня мерж в `main` прошёл, но `deploy-prod.yml` **не стартовал** (job «not started») с аннотацией: *«recent account payments have failed or your spending limit needs to be increased»*. Причина — приватный репо + лимит бесплатных минут Actions (конец месяца / способ оплаты). Деплой у нас идёт **только через Actions** (прямой SSH с машины режётся), так что блок Actions = блок деплоя; код застрял в `main`, на проде его нет, хотя build/гейты зелёные.

**Лечение (выбор владельца):** сделали репозиторий **публичным** → у публичных репо Actions бесплатны и без лимита (секретов в репо нет — только `.env.example`; предварительно прогнал скан истории/grep на ключи — чисто). Альтернатива — поднять spending limit / починить оплату. После публикации — `gh run rerun <id>` → деплой прошёл.

**Почему важно для brain/GONBA:** GONBA — тоже приватный репо с деплоем через Actions. Тот же риск (исчерпание минут под конец месяца особенно), тот же диагноз («job not started» с биллинг-аннотацией, НЕ ошибка кода — легко увести в отладку сборки). Стоит знать заранее: либо публичный репо, либо следить за минутами/лимитом. Прямой SSH-деплой как запасной путь у нас отрезан (G8), у GONBA — проверить, есть ли fallback.
