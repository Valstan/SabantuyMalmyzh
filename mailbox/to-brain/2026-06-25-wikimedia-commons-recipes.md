# Wikimedia Commons: батч-rate-limit + API-сверка лицензий (переносимо)

Контекст: добирал свободные картинки для игры «Угадай по картинке» (заменить
неудачный кадр + 2 новых вопроса). Два момента оказались неочевидными и
переносимыми на любой Payload+Next контент-сайт, тянущий картинки с Commons
(кандидат — **GONBA**, тот же стек, контент-сайт).

## Грабля 1 — Commons 429 на пачке скачиваний: IP-wide и держится МИНУТАМИ

Скрипт качал ~10 файлов с `upload.wikimedia.org` подряд без пауз → словил
**HTTP 429 Too Many Requests**. Неочевидное:

- Лимит **IP-wide и долгий**: после серии запросов throttle висит **минуты**, а
  не секунды. Per-request retry с экспоненциальным backoff В ПРЕДЕЛАХ одного
  прогона (15→30→…→90 c) **частично не помог** — часть файлов так и не дошла.
- Усугубляет то, что я перед этим дёргал Commons **API** (сверка лицензий) +
  качал превью на визуальную проверку → бюджет уже был подъеден.

Что сработало:
1. **Пауза между запросами** (`sleep ~1.2 c`) + ретрай с backoff на 429/5xx —
   в `process-quiz-images.mjs`. Снимает «пачку», но не спасает, если IP уже в
   пеналти.
2. **Реюз уже скачанных байтов**: превью, что я тянул для визуальной проверки,
   переиспользовал как источник финальных файлов (re-encode локально, без сети) —
   так два «непролезших» файла получил без новых запросов к Commons.
3. Растягивать во времени / делать в фоне с длинным cooldown.

Рецепт для REFERENCE/GOTCHAS: *«Commons батч-скачивание → вежливая пауза +
backoff; при IP-throttle (минуты) не долбить, а переиспользовать локальные
копии / разнести по времени»*.

## Рецепт 2 — сверка лицензии/автора/URL по Commons API (не доверять авто-подбору)

Чтобы не закоммитить картинку с неверной лицензией/битым URL, сверял КАЖДЫЙ
финальный файл по API (а не по тому, что «нашлось»):

```
GET https://commons.wikimedia.org/w/api.php
  ?action=query&format=json
  &titles=File:<Имя>.jpg            # или generator=search&gsrnamespace=6
  &prop=imageinfo
  &iiprop=url|extmetadata|size|mime
  &iiextmetadatafilter=LicenseShortName|Artist
```

Отдаёт: `imageinfo[0].url` (прямой upload-URL — кладёшь в манифест без угадывания
хэша), `extmetadata.LicenseShortName.value` (лицензия), `extmetadata.Artist`
(автор для атрибуции). Поиск файла — `generator=search&gsrnamespace=6` (file-ns),
сортировка по разрешению. Кириллические имена файлов в shell бьются (cp1251) —
надёжнее искать по ASCII-запросу и матчить нужный title в ответе.

Брал только CC0 / CC-BY / CC-BY-SA / PD; **чужие-festival кадры отбраковывал**
(масленичный столб, шотландский канат для игры про Сабантуй — визуально
«не та» обстановка → новый mismatch, хуже, чем отсутствие вопроса). И финально —
**визуальная проверка** каждого файла (Read jpg напрямую / sharp→PNG), родня
G44/G82: имя/категория ≠ пригодность кадра.
