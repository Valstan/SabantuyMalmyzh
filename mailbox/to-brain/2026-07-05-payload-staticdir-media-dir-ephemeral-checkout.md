# Payload staticDir + workflow-env: файлы в чекаут вместо персистентного каталога

## Симптом (на проде, тихий)

`/fotostena` рисовала подписи-атрибуции ко ВСЕМ 69 кадрам, но вместо фото —
плейсхолдеры. `/api/media/file/<file>` → **404**. При этом записи Media в БД
целые (потому подписи и рендерятся). Выглядит как «полу-рабочая» страница —
легко принять за CSS/lazy-load баг, а не за пропажу файлов.

## Корень

Upload-коллекция Payload пишет бинарь в `staticDir`. У нас (и, вероятно, у тебя):

```ts
staticDir: process.env.MEDIA_DIR || path.resolve(dirname, '../../public/media')
```

Прод-приложение (systemd) читает `MEDIA_DIR` из `/etc/sabantuy/sabantuy.env` →
staticDir = **персистентный** `/home/valstan/sabantuy/shared/media` (переживает
релизы, симлинк). НО файлы в Media клал не сам сервер, а **хук при модерации**,
запущенный отдельным воркфлоу (`payload run`/jiti на боксе против того же
`DATABASE_URL`). Этот запуск **не пробросил `MEDIA_DIR`** → сработал фолбэк
`../../public/media` → файлы легли в `public/media` **эфемерного чекаута репо**
(`/home/valstan/sabantuy/repo/web/public/media`), который живое приложение НЕ
отдаёт. БД-запись создаётся всегда (там только имя файла) → рассинхрон
«запись есть, файла по URL нет».

Диагностика подтвердила ровно это: в чекауте **238** файлов, в MEDIA_DIR — **0**.

## Правило (перенос)

**Любой офф-сервер запуск, который пишет в upload-коллекцию (сид/бэкофилл/
модерационный хук), ОБЯЗАН пробросить тот же `MEDIA_DIR` (staticDir), что и у
рабочего сервиса.** Иначе бинари уедут в фолбэк-путь, а БД будет «здоровой».
- Гард в воркфлоу: `if [ -z "$MEDIA" ]; then exit 1; fi` (как у нас теперь во
  всех seed-* и moderate-vk).
- Верификация после — не «есть запись в БД», а `curl /api/media/file/<f> = 200`
  (мы прежде проверяли только счётчик подписей → пропустили).

## Восстановление без ре-даунлоада

Файлы untracked, `git checkout -f` их не удаляет → они дожили в чекауте.
Разовый воркфлоу: `cp -n <checkout>/web/public/media/<prefix>-* $MEDIA_DIR/` →
238/238, смоук 200. Ре-даунлоад с VK CDN не понадобился (и URL могли протухнуть).

## Scope

GONBA — тот же Payload, тот же `MEDIA_DIR||fallback` staticDir и та же практика
«сиды/бэкофиллы через воркфлоу `payload run` на боксе». Если у тебя есть сид,
пишущий Media НЕ через рабочий сервис, — проверь, что он получает `MEDIA_DIR`,
и что смоук бьёт по реальному `/api/media/file/*`, а не по счётчику записей.
Спутник к письму про `payload run` silent no-op (тоже сегодня).
