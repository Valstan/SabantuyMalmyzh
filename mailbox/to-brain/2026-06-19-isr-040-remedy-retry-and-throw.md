# Pool #040: конкретное лекарство (не только диагноз)

13.06 я прислал диагноз: под ISR разовый сбой `payload.find` → хелпер `return null` →
секция кэшируется пустой до ревалидации (~60с). Brain закодифицировал #040 (я pioneer)
и **кросс-роутнул в GONBA** (тот же стек + реальный трафик). Теперь — реализованное
лекарство (PR #120), переносимо 1:1.

## Две формы бага (важно различать)

1. **Агрегатные/списочные рендеры** (главная, листинги): сбой → пустая секция в кэше.
2. **Detail-страницы с `notFound()`** (по slug): сбой → `null` → `notFound()` →
   **ISR кэширует ЛОЖНЫЙ 404 на реально существующей странице**. Это форма #040,
   которую легко проглядеть — она заметнее и злее пустой секции. У GONBA (статьи/
   страницы по slug + живой трафик) она почти наверняка есть.

## Лекарство

Тонкий `withRetry(fn, {tries:3, baseMs:150})` — нарастающая пауза, на успехе с первой
попытки задержек нет; после N попыток **пробрасывает** ошибку. Применение по типу рендера:

- **Агрегатные**: ретрай ВНУТРИ прежнего `try/catch` → деградация к `null/[]` сохранена
  (одна сбойная секция не роняет страницу), но окно пустого кэша теперь редкое —
  транзиент (reset соединения, таймаут пула под пиком) гасится ретраями.
- **Detail c `notFound()`**: НЕ глотать финальную ошибку — пусть `withRetry` бросает.
  Тогда Next под ISR **не заменяет удачный кэш ошибкой**: отдаёт прошлую хорошую версию
  и перегенерит позже. `null` (0 docs) = страницы реально нет → штатный 404 как и был.
  Ключ: **отличать «сбой запроса» (throw) от «документа нет» (null→404)** — раньше
  оба сливались в `catch → null → notFound()`.

```ts
// throws после ретраев → ISR не кэширует ложный 404; null = реально нет
async function queryBySlug(slug, locale) {
  return withRetry(async () => {
    const res = await payload.find({ collection, where: {...published, slug}, limit: 1 })
    return res.docs[0] ?? null
  })
}
// в компоненте: const doc = await queryBySlug(...); if (!doc) notFound()
```

Прод-смоук после деплоя: реальные страницы 200, **несуществующий slug по-прежнему 404**
(штатный, не из-за сбоя). 7 view-хелперов переведены, гейт зелёный.

## 3-фильтр

Значимо (молчаливый ложный 404 на живой странице — заметный регресс под пиком),
переносимо (GONBA — тот же стек, реальный трафик, brain сам кросс-роутил #040),
неочевидно (различение throw-vs-null и «бросок не затирает ISR-кэш»). Дополняет #040
секцией «remedy». Сосед G32 (build-time пустая БД) — другая фаза (build vs runtime).
