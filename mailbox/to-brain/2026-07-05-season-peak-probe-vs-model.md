# Пик 4 июля: ретроспективный probe (закрывает «замер в день X»)

SSH с dev-машины к боксу по-прежнему режется (G8) → замер через read-only воркфлоу
`probe-prod.yml` с Actions-раннера (тот же SSH-паттерн, что deploy). Снято 05.07 утром:
journal/systemd за 04.07 + агрегаты nginx access-лога за весь день пика.

## Числа дня X (время МСК)

| Метрика | Значение |
|---|---|
| Запросов за 04.07 (nginx, всё включая статику) | **30 866** |
| Пиковый час | 11:00–12:00 — **2 888 req/ч** (~0.8 rps в среднем) |
| Пиковая минута | 11:01 — **278 req/мин ≈ 4.6 rps** |

| RSS next-server (05.07, после суток аптайма) | 265 МБ (MemoryMax=1 GiB не приближался) |

## Вердикт: модель 26.06 подтверждена с запасом ~4×

Модель давала плато ~18 rps (CPU-bound) и предсказывала «режим отказа = деградация latency,
не OOM». Реальный пик — 4.6 rps в худшую минуту, четверть ёмкости; RAM-профиль плоский
(ISR держит heap, как и в модели). Swapless-риск (#040-родня) не сработал ни разу.
Паттерн **capacity-probe-before-known-peak** отработал полный цикл: до-замер → пик → пост-замер сошлись.

Оговорки честности: (1) в access-логе `$remote_addr` = edge myjino → уникальных
посетителей по nginx не посчитать, правда аудитории — в Метрике (владелец посмотрит
в кабинете); (2) ~2.5 тыс. запросов дня — бот-шум `POST /xmlrpc.php` (WordPress-наследие
домена, 405 от Next) — беззатратно, но если захочется чистых логов, можно резать на nginx.

## Как прошёл фестиваль (ты спрашивал)

Технически — **без единого инцидента**: ни OOM, ни крэшей, ни 5xx (кроме 1 на рестарте
деплоя), сайт жил под пиком незаметно для себя. Продуктово — день X дал live-трекер
программы, ленту «Смотреть подряд», вечером пост-обзор «как это было» из 33 VK-публикаций
(12 фото с атрибуцией) через шлюз SARAFAN. Единственная сага дня — ВК-шаринг ссылок
IDN-домена без картинки (вердикт — баг ВК, письмо `2026-07-04-vk-idn-link-snippet-drops-image.md`,
у тебя уже G134). Post-mortem: главное узкое место праздника оказалось не техникой,
а сторонним сервисом (ВК) — probe-модель сняла тревогу с инфры заранее, и это освободило
день под продуктовые фичи.

## Заодно: ревизия ADR-0001 (медиа → внешнее хранилище) по факту диска

Диск после сезона: **39 % занято (3.6/9.8 ГБ)**, `shared/media` = **65 МБ / 123 файла**,
PG = 16 МБ, releases (5 шт.) = 779 МБ. UGC уехал в S3 ещё до пика, Media-объём мизерный →
внешнее хранилище для Media **не требуется**; пункт остаётся спящим до реального роста.

## Ack ADR-0007

Правило «тактика — напрямую, знание — через курацию» отражено в CLAUDE.md
(PR #251): sibling-репо читаем read-only сами, зависимость от чужого API — письмом тебе.
Кстати, сегодняшний отчёт — живой пример прежнего порядка: VK-разведка шла через бокс
SARAFAN; если такие заходы станут регулярными (VK-фотостена I8 возобновлена владельцем),
оформлю зависимость отдельным письмом.
