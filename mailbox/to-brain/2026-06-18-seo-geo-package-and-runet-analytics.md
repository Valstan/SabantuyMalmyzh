# Находка: пакет «SEO + выдача в нейросетях (GEO)» для Payload+Next

Внедрил на Сабантуе по заявке владельца («счётчики + СЕО, особенно под ответы
нейросетей»). Стек наш = стек GONBA → переносимо 1:1. Делюсь как pool-идею.

## 1. GEO ≠ классическое SEO (актуально на 2026)

Нейросети (ChatGPT/Perplexity/Gemini/Copilot) **не ранжируют — цитируют**. Чтобы
попасть в ответ, контент должен быть машиночитаемым и фактологичным. По обзорам
2026 структурированная разметка повышает «обнаружимость» у LLM заметно (порядок
+50–70%), а definition-first абзацы и Q&A-блоки — ещё. Ключевое практичное:

- **Schema.org JSON-LD** как **серверные** компоненты → **ноль JS для браузера**
  (важно для нашего лёгкого VPS). У меня: `WebSite`+`Organization` (глобально в
  layout), `Event`/`Festival` (главная — дата/место/что), **`FAQPage`** (выверенные
  Q&A — их LLM цитирует охотнее всего), `BreadcrumbList` (страницы). Билдеры в
  `lib/jsonLd.ts`, инъекция — крошечный `<JsonLd>` с `dangerouslySetInnerHTML` +
  экранирование `<` → `<`.
- **`/llms.txt`** (стандарт llmstxt.org) — карта сайта для ИИ-краулеров: H1 +
  blockquote-сводка + список ключевых ссылок с фактами. Роут в корне `app/`
  (НЕ в route-group — та же грабля, что robots/sitemap, наш PR #6).
- **robots** — ЯВНО пускать ИИ-ботов (`GPTBot`, `OAI-SearchBot`, `ClaudeBot`,
  `PerplexityBot`, `Google-Extended`, `YandexBot`…). Для GEO они нужны; многие
  сайты их по привычке режут — это анти-GEO.
- **Единый источник фактов** (дата/место/название в одном конфиге) → разметка,
  llms.txt и контент не разъезжаются. Для образовательного/локального проекта
  это ещё и про доверие (наш фактчек-этос из игры-угадайки).

## 2. Env-gated отложенная аналитика (паттерн)

Счётчики (Метрика/LiveInternet) — `next/script` (`afterInteractive`/`lazyOnload`,
вне критич. пути) + **гейт по env**: id пуст → компонент не рендерит ничего. Плюсы:
в dev и до настройки прода трафик не пачкается; `NEXT_PUBLIC_*` бейкается из CI
`vars.*` (как наш `NEXT_PUBLIC_SERVER_URL`), владелец вставляет id один раз. Плашка
согласия (152-ФЗ + ToS Яндекса) — клиентский компонент, показывается только если
счётчики включены, не блокирует, выбор в localStorage.

## 3. Свежая Рунет-интел (проверял веб 2026-06-18)

- **Top.Mail.ru / Рейтинг Mail.ru: с 27.03.2026 НЕЛЬЗЯ добавлять новые сайты.**
  Если у GONBA в планах был Top.Mail.ru как второй счётчик — отпал. Живые пары
  сегодня: **Яндекс.Метрика** (главная, влияет на ранжирование в Яндексе) +
  **LiveInternet** (лёгкий, публичный рейтинг Рунета) или GA4.

## Фильтр

Значимо (видимость в поиске и в ИИ-ответах — растущий канал), переносимо (GONBA на
том же стеке, тоже контент-сайт), неочевидно (GEO-фрейминг, llms.txt, env-gated
аналитика, Top.Mail.ru-факт). Кандидат в `cross-project-ideas/` (pool) и, возможно,
в `REFERENCE.md` как рецепт «GEO-пакет для Payload+Next».
