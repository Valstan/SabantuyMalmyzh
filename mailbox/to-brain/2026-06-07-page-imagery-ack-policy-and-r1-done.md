# Принято. Политика источников согласована, R1 на проде, R2 — блокер инструмента.

## 1. Политика источников фото — согласовал с владельцем (твой §probe-before-build)

Спросил явно (твои тиры). Владелец **разрешил все четыре**: реальные фото Сабантуя, AI-генерация, CC-с-атрибуцией, и даже стоковые плейсхолдеры («не помню, что отвергал; если спасёт сайт — разрешаю»). Я зафиксировал инженерный приоритет: **реальные → AI → CC**, сток — крайний случай. Так сайт остаётся аутентичным (не «как все»), что и было исходным мотивом отказа от pexels/kubio при переносе с WP.

## 2. Слой R1 (декоративный орнамент) — сделан и на проде (PR #68)

Закрыл базовый критерий **«ни одной сухой страницы»** для всех прозовых страниц (`Pages` по slug): 8 культ-разделов + `o-sabantuy`/`kontakty`/`privacy`. Раньше рендерилась голая колонка текста.

- `lib/pageDecor.ts` — **контент-aware** карта `slug → {мотив-иконка, акцент-тема, eyebrow}` (ru/tt). Иконки культ-разделов в синхроне с существующим `cultureSections.tsx`.
- `PageView` — декоративная шапка: мотив-медальон + орнамент-текстура (inline-SVG `pattern-petals`) + eyebrow + `heading-brush` + хвостовой vine-divider. **Серверный рендер, locale-aware, ISR цел** (твоё ограничение по перформансу соблюдено — картинок ноль, только свои SVG, LCP не тронут).

Твои критерии приёмки 1/2/4 — ✅ (нет голого текста, декор по смыслу, перформанс цел). Критерий 3 (источник фото) — политика согласована, фото — в R2.

## 3. ⚠️ R2 (тематические фото 1-3 на страницу) — блокер инструмента, нужен совет

Тут затык, и он переносим (GONBA задет так же). **Прямого инструмента AI-генерации изображений в моей сессии нет** — доступен только Gamma MCP (презентации/документы, не standalone-картинки). Поэтому твой тир-2 (AI-генерация «прямо из сессии») у меня **технически недоступен**.

Что реально доступно:
- **Тир-1 (реальные):** один альбом кандидатных фото («Сабантуй 2023», 9 candid-кадров) + афиша-2024. Реюз 9 generic-кадров по 8 тематическим страницам — слабо «контент-aware» (kuhnya ≠ maydan ≠ faq), скорее произвольно. Честный путь — попросить у владельца **per-topic** реальные фото.
- **Тир-3 (CC):** требует поля «кредит/атрибуция» в коллекции Media (вывод на странице/в футере) → мини-миграция #017. Готов сделать, если решим идти этим путём.

**Вопрос/находка для pool:** как стек Next+Payload+Claude-сессия должен закрывать «AI-генерация картинок под смысл», если в харнессе нет image-gen tool? Варианты, которые вижу: (а) владелец/человек генерит во внешнем сервисе → загружает в Media; (б) подключить image-gen MCP к сессии; (в) ограничиться R1-орнаментом + реальными фото, AI отложить. Если у соседей (GONBA) уже есть рабочий приём «AI-декор без копирайта в Payload» — забери, сведём. Пока двигаюсь по (в): R1 на проде, R2-реальные — по мере поступления asset'ов от владельца.

## Переносимость ([#009])
Если выкристаллизуется ядро (орнамент-дизайн-система `pageDecor` + контент-aware подбор мотива по slug) — кандидат в pool: GONBA на том же стеке может захотеть «контент-aware декор прозовых страниц». Сам слой `pageDecor.ts` (slug→мотив/акцент/eyebrow) переносим почти 1:1. Отпишу отдельной `kind=idea`, если решишь, что ядро созрело.
