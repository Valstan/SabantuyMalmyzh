# Идея: команда `/obriv` — восстановление после обрыва связи

## Контекст / проблема

При обрыве интернета/сессии посреди работы пользователь сейчас вынужден **руками писать длинное объяснение**: «был обрыв, проверь не потерял ли ты работу, перепроверь себя и продолжай». Это рутина, тратит время и внимание на каждом обрыве (а они случаются регулярно на этой инфраструктуре).

**Владелец (@valstan) просит сделать это кросс-проектным мандатом** — раскатать команду `/obriv` во все управляемые проекты, чтобы после обрыва он просто вводил `/obriv`, а агент сам себя перепроверял и продолжал. Цитата: «очень нужная удобная фича будет, ускоряющая работу».

## Что делает `/obriv`

Агент по одной короткой команде:
1. **Реконструирует состояние из ground truth** (не из памяти): `git status/branch/log/stash`, открытые PR (`gh`), фоновые задачи и их `*.output`, занятые порты.
2. **Сверяется с PR**, чтобы не задвоить уже сделанное (создание PR, коммиты).
3. **Проверяет целостность недавно записанных файлов** — на нашей машине обрыв иногда совпадает с битой записью: лишний `NUL`-байт / UTF-16 / BOM → git показывает исходник как `Bin`, хотя сборка может «проглотить». Чистка NUL + пересохранение UTF-8. *(Это реальная грабля, поймана сегодня: при правке `ScheduleList.tsx` в строковый литерал попал `` вместо пробела — файл стал бинарным для git. Кандидат в GOTCHAS отдельно, если хочешь.)*
4. **Реконсилирует последнее намеренное действие** с фактом (правка легла? если не уверен — перечитать файл, не доверять памяти).
5. **Перепрогоняет гейты**, если трогался код (typecheck/lint/build).
6. **Докладывает** компактную сводку «что цело / что потеряно» и **продолжает прерванную нить**.

Ключевой принцип: **idempotency — не переделывать вслепую уже сделанное**.

## Почему мандат (3-фильтр пройден)

- **Значимость:** убирает повторяющееся ручное трение на каждом обрыве у владельца.
- **Переносимость:** механика 100% стек-агностична (git/gh/файлы); проектное — только команда гейтов (`corepack pnpm -C web …` у нас vs `npm`/иное у других).
- **Неочевидность:** без явной команды агент после обрыва склонен либо переспрашивать, либо переделывать работу заново → потеря/задвоение.

## Портативный спек (адаптировать под каждый проект)

Наш файл: [`.claude/commands/obriv.md`](../../.claude/commands/obriv.md) (этот репо, эта же ветка/PR). Для раскатки в проект меняется только:
- путь/команда гейтов (typecheck/lint/build) под стек проекта;
- ссылка на проектный `SESSION_HANDOFF` (или аналог) для образца smoke-проверок.

Всё остальное (git/gh/NUL-чистка/реконсиляция/доклад) переносится 1:1.
