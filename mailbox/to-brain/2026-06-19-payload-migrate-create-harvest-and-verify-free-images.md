# Две переносимые находки (Payload-миграции + свободные картинки)

Контекст: добавляли в `quiz-questions` поля `game`(enum)/`image`/`image_source` + `game` в `quiz_results` (versioned-коллекция). GONBA — тот же стек (Payload+Postgres, hand-authored `.ts`+`.sql` миграции, G6/G7/G28).

## 1. `payload migrate:create` без снапшот-baseline печатает ПОЛНУЮ схему

При `push:true` в dev и отсутствии выровненного снапшота `payload migrate:create` сгенерил миграцию, **создающую ВСЕ таблицы/энумы с нуля** (CREATE TABLE на весь проект), а не инкремент `ADD COLUMN`. Накатывать такое на прод нельзя (таблицы уже есть, без `IF NOT EXISTS`).

**Рабочий приём:** использовать вывод `migrate:create` как **справочник точных Drizzle-имён** (колонки версионной таблицы `version_<field>`, энумы `enum_<coll>_<field>` и `enum__<coll>_v_version_<field>`), затем **руками** написать инкрементную миграцию (`ALTER TABLE … ADD COLUMN IF NOT EXISTS` + `DO/EXCEPTION` для `CREATE TYPE`) + `.sql`-зеркало в стиле репо, удалить сгенерённый файл и его `.json`-снапшот, вернуть `index.ts` к ручному стилю. Так точные имена — от инструмента, а идемпотентность/прод-безопасность — наши. (Доп.: `ADD COLUMN … DEFAULT 'x'` в Postgres **бэкафиллит существующие строки** дефолтом сразу — отдельный `UPDATE` для бэкафилла избыточен, но не вреден.)

## 2. Свободные картинки Commons — верифицировать ВИЗУАЛЬНО до использования

Для игры «угадай по картинке» брали изображения с Wikimedia Commons (CC0/CC-BY/CC-BY-SA/PD) через API (`generator=search` + `iiprop=url|extmetadata` → URL+лицензия одним запросом; `prop=imageinfo` по `titles=` батчем, чтобы не словить 429). **Грабля:** поиск по релевантности шумит — CC0-файл «Tournoi de Kurash 03» (по запросу про корэш) оказался кадром с судьёй на ковре в каком-то африканском зале, БЕЗ борьбы. **Перед использованием каждую картинку прогнали `sharp`→`Read` (G44) и посмотрели глазами** — отбраковали, заменили на Sabantuy-köräş. Лицензию/автора/страницу-источник (`descriptionurl`) сохранили для атрибуции; показываем ссылку под фото. Вывод: free-license ≠ годная картинка — визуальная проверка обязательна, имя файла не гарантирует содержимое.

## 3-фильтр

Значимо (миграции versioned-коллекций — частая операция; неверная картинка в образовательной игре бьёт по репутации), переносимо (GONBA — тот же стек, те же миграции; контент-сайт с картинками), неочевидно (full-schema от migrate:create удивляет; релевантность Commons-поиска обманчива). Кандидаты в `GOTCHAS.md` (рядом с G6/G7/G28) и `REFERENCE.md` (рецепт «инкремент-миграция через migrate:create-harvest»).
