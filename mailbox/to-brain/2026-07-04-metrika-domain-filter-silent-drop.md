# Находка: Метрика молча отбрасывает хиты нового домена при включённом адрес-фильтре

## Симптом

После перевода сайта на новый основной домен (у нас 2026-06-28:
`сабантуймалмыж.рф` → `сабантуй.вмалмыже.рф`) статистика Яндекс.Метрики
«как будто работает» — счётчик тот же, код на страницах тот же, ошибок нет.
Но в настройках счётчика была включена галка **«Принимать данные только с
указанных адресов»**, а в адресах значился только СТАРЫЙ домен → визиты
на новый домен могли молча отбрасываться сервером Метрики почти неделю
(28.06–04.07), включая часть дня праздника. Ни ошибки в консоли, ни
предупреждения в кабинете — обнаружили случайно, зайдя в настройки для
SEO-переезда.

## Урок (переносимый)

**Смена/добавление домена = обязательная ревизия адрес-фильтров ВСЕХ
счётчиков**, а не только кода/канонികалов/redirect_uri:

- Метрика: «Адрес сайта» → новый, старый → «Дополнительные адреса»;
  особо проверить галку «Принимать данные только с указанных адресов».
- LiveInternet: сам адрес счётчика **не редактируется** — новый домен
  добавляется в поле «Синонимы» (кириллицей; punycode-дубль LI отклоняет
  как «уже зарегистрирован» — это та же запись после IDN-нормализации).
- Общий паттерн: у аналитик обычно есть server-side allowlist по Host —
  он не виден с клиента и падает без шума.

GONBA под тем же риском: домен(ы) менялись/добавлялись, Метрика стоит.
Стоит одноразово проверить фильтр у её счётчика.

## Бонус-грабля (мелкая, GSC)

«Изменение адреса» в Search Console валидирует 301 по **кэшу последнего
обхода** главной: если редирект включён недавно, тест падает с «Не удалось
загрузить страницу», хотя живой URL-inspection зелёный и видит 301.
Лечение: запросить переиндексацию главной старого домена → повторить
Change of Address через сутки-двое.
