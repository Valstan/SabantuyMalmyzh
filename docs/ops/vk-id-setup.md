# Регистрация приложения VK ID (вход через ВКонтакте) — для владельца

> Разовая настройка, ~15–20 мин. Нужен аккаунт ВКонтакте (лучше — администратора сообщества праздника).
> По итогу: **ID приложения** (публичный — можно прислать в чат) и **Защищённый ключ** (секрет — через
> зашифрованный секрет GitHub, **НЕ в чат**, #008). После этого я подключу вход «Войти через VK» на сайт.

## Зачем

Чтобы посетители могли войти на сайт одним нажатием через ВКонтакте. Тогда их фото/комментарии/лайки
в «Народной ленте» закрепляются за аккаунтом — управлять «своим» можно с любого устройства (а не только
с того телефона, где выложил), и под публикацией показывается имя/аватар из VK. Анонимный режим
останется: войти — по желанию.

## Что в итоге понадобится мне

| Значение | Где взять | Как передать |
|---|---|---|
| **ID приложения** (App ID, число) | карточка приложения VK | можно прямо в чат (не секрет) |
| **Защищённый ключ** (Client Secret) | карточка приложения → «Безопасность» | **через секрет GitHub** (см. §4), не в чат |

## Шаги

### 1. Создать приложение

- Открыть **VK ID для разработчиков**: https://id.vk.com/about/business/go (или https://vk.com/apps?act=manage → «Создать приложение»).
- Войти под аккаунтом ВКонтакте.
- **Создать приложение** → тип/платформа: **«Веб»** (веб-сайт). Не «Мобильное», не «VK Mini Apps».
- Название: например `Сабантуй Малмыж`. Иконка — по желанию (можно логотип праздника).
- Согласиться с правилами платформы VK ID.

### 2. Указать домен и адрес возврата (redirect)

В настройках приложения (раздел «Настройки» / «Платформы» / «Доверенные redirect URL» —
точные названия меняются в интерфейсе VK):

- **Базовый домен / Domain:**
  ```
  xn--80aaac1aqpgcf4bqn1j.xn--p1ai
  ```
  (это «сабантуймалмыж.рф» в техническом виде — punycode; VK обычно требует именно его).
  Если поле принимает кириллицу — можно добавить и `сабантуймалмыж.рф`.

- **Доверенный redirect URL / Trusted Redirect URI** — добавить ровно этот адрес:
  ```
  https://xn--80aaac1aqpgcf4bqn1j.xn--p1ai/api/auth/vk/callback
  ```
  Если VK позволит — добавьте и кириллический вариант `https://сабантуймалмыж.рф/api/auth/vk/callback`.
  ⚠️ Адрес должен совпадать **символ в символ** — это критично для безопасности входа.

### 3. Забрать ключи

В карточке приложения (раздел «Настройки» / «Безопасность»):

- **ID приложения** (App ID) — число вида `5184…`. → пришлите мне в чат.
- **Защищённый ключ** (Client Secret / «Защищённый ключ») — длинная строка. → **в чат не присылать**, положить в секрет GitHub (§4).
- Если есть **«Сервисный ключ доступа»** — пока не нужен, но запишите себе.

### 4. Передать секрет безопасно (как делали для S3)

Защищённый ключ — это секрет. Кладём его в зашифрованные секреты репозитория (я их вижу только внутри
сборки, не в открытом виде):

1. Открыть https://github.com/Valstan/SabantuyMalmyzh → **Settings** → **Secrets and variables** → **Actions**.
2. **New repository secret**:
   - Name: `VK_CLIENT_SECRET`
   - Secret: вставить Защищённый ключ → **Add secret**.
3. **ID приложения** (публичный) можно прислать в чат — я добавлю его как переменную сборки
   `NEXT_PUBLIC_VK_APP_ID` (Variables, не секрет).

После этого я разовым воркфлоу (по образцу `apply-s3-keys.yml`) допишу ключ в прод-окружение и включу
кнопку «Войти через ВКонтакте». Никакие пароли/секреты в чат и в репозиторий не попадают.

## Заметки

- **Domain в punycode** — у нас был ровно такой нюанс с CORS бакета: браузер для кириллического домена
  работает через `xn--…`. Поэтому базовый домен и redirect задаём в punycode.
- **Что увидит посетитель:** на телефоне с установленным приложением VK — обычно вход в один тап;
  на остальных — стандартное окно подтверждения VK ID.
- **Объём доступа (scope):** нам нужен только базовый профиль (имя и аватар). Email — по желанию;
  если не хотите запрашивать email, ничего дополнительно включать не надо.
- **Приватность (152-ФЗ):** мы храним из VK только идентификатор, имя и ссылку на аватар — для подписи
  публикаций и привязки «моего» контента. Это отражу в Политике обработки ПДн.
