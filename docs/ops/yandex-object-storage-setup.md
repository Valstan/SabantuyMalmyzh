# Настройка Yandex Object Storage для «Народной ленты» (для владельца)

> ✅ **Сделано 2026-06-27.** Бакет `sabantuy-malm` активен, ключи в прод-env, CORS под боевой домен,
> e2e зелёный. Документ оставлен как референс/для ротации ключей. Ниже — фактический рабочий процесс.
>
> Разовая настройка, ~20–30 мин в консоли Яндекс Облака. Нужен вход в Яндекс Облако + платёжный
> аккаунт. **«Бесплатный тариф» отдельно включать НЕ надо** — нет такой кнопки; баланс на платёжном
> аккаунте + бесплатная квота Object Storage применяются сами, сверх — копейки со счёта. После шагов
> — **имя бакета** (не секрет) мне, а **ключи** — через зашифрованные секреты GitHub (см. §6, НЕ в чат, #008).

## Зачем
Фото/видео ленты хранятся в Object Storage, браузер тянет их напрямую из Яндекса (наш слабый бокс
не грузится в пик). Загрузка — браузер кладёт файл прямо в бакет по временной подписанной ссылке.

## Шаги

### 1. Аккаунт и платёж
- Войти в **Яндекс Облако**: https://console.yandex.cloud
- Создать **платёжный аккаунт** (привязать карту/баланс). Новым обычно дают **грант ~₽4000 на 60 дней** —
  фестиваль, скорее всего, уложится в него.
- В облаке уже есть каталог (folder) по умолчанию — он подойдёт.

### 2. Бакет
- Консоль → **Object Storage** → **Создать бакет**.
  - Имя: например `sabantuy-ugc` (глобально уникально, латиница/цифры/дефис).
  - Класс хранилища: **Стандартное**.
  - Доступ: **Публичный на чтение объектов** (Read для объектов = «Публичный»), чтобы лента показывала медиа.
    Запись/листинг — оставить приватными.
- Запомни имя бакета — пришлёшь мне.

### 3. CORS бакета (чтобы браузер мог загружать файл напрямую)
- Бакет → вкладка **Безопасность → CORS** → добавить правило:
  - **Origin:** ⚠️ **ВАЖНО — точный punycode-apex `https://xn--80aaac1aqpgcf4bqn1j.xn--p1ai`**
    (плюс при желании `https://*.xn--80aaac1aqpgcf4bqn1j.xn--p1ai` для поддоменов).
    **Грабля:** правило вида `https://*.домен` (wildcard-поддомен) **НЕ покрывает «голый» apex**, а
    браузер для IDN-домена (`.рф`) шлёт в `Origin` именно **punycode-apex** → preflight `OPTIONS` → 403
    → загрузка из браузера падает. Cyrillic-форму в CORS добавлять не нужно (в `Origin` её не бывает).
    Проверка без браузера: `curl -i -X OPTIONS -H "Origin: https://xn--…xn--p1ai" -H "Access-Control-Request-Method: PUT" <publicUrl>` → должен быть `Access-Control-Allow-Origin`.
  - **Methods:** `GET, PUT, HEAD`
  - **Headers:** `*`
  - **Max age:** `3600`

### 4. Сервис-аккаунт (робот, под которым сайт пишет в бакет)
- Консоль → **Сервисные аккаунты** → **Создать**.
  - Имя: `sabantuy-ugc-writer`.
  - Роль на каталог/бакет: **`storage.editor`** (читать+писать). Можно сузить до `storage.uploader`+`storage.viewer`.

### 5. Статический ключ доступа (S3-ключ)
- Открыть сервис-аккаунт → **Создать новый ключ** → **Статический ключ доступа**.
- Получишь пару: **Идентификатор ключа** (`key_id`) и **Секретный ключ** (`secret`).
  ⚠️ **Секрет показывается один раз** — сохрани его в менеджере паролей.

### 6. Передача ключей в прод (фактический способ — через секреты GitHub, без чата)
- **Не секрет** (можно в чат): имя бакета (`sabantuy-malm`), endpoint (`https://storage.yandexcloud.net`),
  регион (`ru-central1`).
- **Секреты (key_id + secret) — НЕ в чат, НЕ в репо.** Способ доставки (использован 2026-06-27):
  1. Владелец кладёт их в **зашифрованные секреты GitHub**: репо → **Settings → Secrets and variables →
     Actions → New repository secret** → `S3_ACCESS_KEY_ID` и `S3_SECRET_ACCESS_KEY` (только браузер).
  2. Запускаем разовый воркфлоу **`apply-s3-keys.yml`** (`gh workflow run apply-s3-keys.yml --ref main`):
     он по SSH дописывает в `/etc/sabantuy/sabantuy.env` 6 строк (4 литерала + 2 из секретов) и
     рестартит сервис. Имя бакета/endpoint/регион/public-base-url — литералами в воркфлоу.
  3. **Ротация ключа:** обновить те же секреты GitHub → перезапустить воркфлоу.
- Итоговый блок в прод-env (для справки; пишет воркфлоу, руками не нужно):
  ```
  S3_ENDPOINT=https://storage.yandexcloud.net
  S3_REGION=ru-central1
  S3_BUCKET=sabantuy-malm
  S3_ACCESS_KEY_ID=<key_id>
  S3_SECRET_ACCESS_KEY=<secret>
  S3_PUBLIC_BASE_URL=https://sabantuy-malm.storage.yandexcloud.net
  ```
- ⚠️ Локально S3-ключи НЕ ставим (`web/.env` без них) — чтобы dev не писал в боевой бакет; локально
  лента работает в degraded-режиме (sign-upload 503), это норма.

## Проверка (я сделаю после получения ключей)
Тестовая presigned-загрузка пробного файла → объект появляется в бакете → публичный URL открывается в браузере.

## Стоимость (ориентир, проверь актуальные тарифы)
Хранение ~₽2/ГБ·мес (для ленты — копейки). Исходящий трафик ~₽1/ГБ (главная переменная; держим в узде:
без автоплея видео, постеры-превью, лимит длины ролика, ленивая подгрузка). Для регионального праздника —
ориентир сотни ₽, часто в пределах гранта.
