# Настройка Yandex Object Storage для «Народной ленты» (для владельца)

> Разовая настройка, ~20–30 мин в консоли Яндекс Облака. Я (Claude) сделать это за тебя не могу —
> нужен твой вход в Яндекс Облако + платёжный аккаунт. Ниже — точные шаги. После них пришли мне
> **имя бакета** (не секрет), а **ключи** положи в прод-env (не в чат, не в репо — #008).

## Зачем
Фото/видео ленты хранятся в Object Storage, браузер тянет их напрямую из Яндекса (наш слабый бокс
не грузится в пик). Загрузка — браузер кладёт файл прямо в бакет по временной подписанной ссылке.

## Шаги

### 1. Аккаунт и платёж
- Войти в **Яндекс Облако**: https://console.yandex.cloud
- Создать **платёжный аккаунт** (привязать карту/баланс). Новым обычно дают **грант ~₽4000 на 60 дней** —
  фестиваль, скорее всего, уложится в него.
- В облаке уже есть каталог (folder) по умолчанию — он подойдёт.

### 2. Бакет
- Консоль → **Object Storage** → **Создать бакет**.
  - Имя: например `sabantuy-ugc` (глобально уникально, латиница/цифры/дефис).
  - Класс хранилища: **Стандартное**.
  - Доступ: **Публичный на чтение объектов** (Read для объектов = «Публичный»), чтобы лента показывала медиа.
    Запись/листинг — оставить приватными.
- Запомни имя бакета — пришлёшь мне.

### 3. CORS бакета (чтобы браузер мог загружать файл напрямую)
- Бакет → вкладка **CORS** → добавить правило:
  - **Origin:** `https://сабантуймалмыж.рф` (и его punycode `https://xn--80aaac1aqpgcf4bqn1j.xn--p1ai`);
    на время разработки можно `*`, потом сузить.
  - **Methods:** `GET, PUT, HEAD`
  - **Headers:** `*`
  - **Max age:** `3600`

### 4. Сервис-аккаунт (робот, под которым сайт пишет в бакет)
- Консоль → **Сервисные аккаунты** → **Создать**.
  - Имя: `sabantuy-ugc-writer`.
  - Роль на каталог/бакет: **`storage.editor`** (читать+писать). Можно сузить до `storage.uploader`+`storage.viewer`.

### 5. Статический ключ доступа (S3-ключ)
- Открыть сервис-аккаунт → **Создать новый ключ** → **Статический ключ доступа**.
- Получишь пару: **Идентификатор ключа** (`key_id`) и **Секретный ключ** (`secret`).
  ⚠️ **Секрет показывается один раз** — сохрани его в менеджере паролей.

### 6. Передача мне
- **Не секрет** (можно в чат): имя бакета, endpoint (`https://storage.yandexcloud.net`), регион (`ru-central1`).
- **Секреты** (НЕ в чат): на проде в `/etc/sabantuy/sabantuy.env`:
  ```
  S3_ENDPOINT=https://storage.yandexcloud.net
  S3_REGION=ru-central1
  S3_BUCKET=sabantuy-ugc
  S3_ACCESS_KEY_ID=<key_id>
  S3_SECRET_ACCESS_KEY=<secret>
  S3_PUBLIC_BASE_URL=https://sabantuy-ugc.storage.yandexcloud.net
  ```
  затем `sudo systemctl restart sabantuy`.
- Для локальной проверки (опц.) те же значения — в `web/.env` (gitignored).

## Проверка (я сделаю после получения ключей)
Тестовая presigned-загрузка пробного файла → объект появляется в бакете → публичный URL открывается в браузере.

## Стоимость (ориентир, проверь актуальные тарифы)
Хранение ~₽2/ГБ·мес (для ленты — копейки). Исходящий трафик ~₽1/ГБ (главная переменная; держим в узде:
без автоплея видео, постеры-превью, лимит длины ролика, ленивая подгрузка). Для регионального праздника —
ориентир сотни ₽, часто в пределах гранта.
