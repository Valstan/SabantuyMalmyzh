# ADR-0001 — Медиа во внешнем хранилище, не self-host с VPS

**Статус:** Accepted (план, частично — MVP локально)
**Дата:** 2026-06-03
**Контекст веха:** M1 → M3

## Контекст

Сайт фестиваля в пик сезона (июнь-июль) отдаёт фото и видео. VPS — небольшой облачный jino (~2 vCPU / 4 GB). Отдавать тяжёлое видео и крупные галереи напрямую с такого сервера в пик = риск исчерпать канал/диск и уронить сайт ровно тогда, когда он нужен.

Соседний проект GONBA уже решил это: медиа синхронизируются на Яндекс.Диск, а отдача идёт через прокси/публичные ссылки (см. `../brain_matrica/cross-project-ideas/REFERENCE.md` R2, и GONBA `web/src/collections/Media.ts`).

## Решение

1. **Фото → внешнее хранилище.** Цель — переиспользовать приём GONBA (Я.Диск): `afterChange`-хук заливает файл во внешнее хранилище, `afterRead` подменяет `url` на внешнюю/прокси-ссылку, локальный оригинал удаляется. Провайдер подтверждается владельцем.
2. **Видео → встраивание плеером** (Rutube/VK), а не загрузка файлов в Payload. В `Gallery.videos[]` хранится только ссылка-embed.
3. **MVP (сейчас):** коллекция `Media` использует локальный `staticDir` (`web/public/media/`, в `.gitignore`). Это временно — до подключения внешнего хранилища (M3). Каркас намеренно оставлен простым, чтобы не тащить интеграцию Я.Диска до того, как поднят прод.

## Последствия

- В `web/next.config.js` уже разрешены remote-хосты `disk.yandex.ru` / `downloader.disk.yandex.ru` для `next/image`.
- При подключении внешнего хранилища: добавить поля синхронизации в `Media` (по образцу GONBA — `yandexPath`, `yandexPublicUrl`, …), хук заливки, прокси-эндпоинт `/api/media/file/[id]`. Это **второй потребитель** паттерна Payload+Media→Я.Диск → повод промотировать REFERENCE R2/R3 в полноценный pool (письмо в `mailbox/to-brain/`).
- Грабли миграций Payload при добавлении полей — см. GOTCHAS G7 (`_v`-таблицы версионируемых коллекций).
