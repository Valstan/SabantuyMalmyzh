# Конвейер AI-картинок для оформления сайта

> Владелец генерит картинки в любой нейросети (Midjourney / Ideogram / Kandinsky /
> Шедеврум / DALL-E — неважно), Claude встраивает их на сайт. Политика источников
> согласована 2026-06-07: AI-изображения разрешены (тир-2), приоритет реальные → AI.

## Процесс (4 шага)

1. **Владелец** даёт задание: «оформи страницу X» / «хочу фон для Y».
2. **Claude** отвечает карточкой задания: для каждой картинки — промт (EN + ru-перевод),
   пропорции (aspect ratio), имя файла.
3. **Владелец** генерит, выбирает лучший вариант, сохраняет в максимальном качестве
   под указанным именем и кладёт в **`design/incoming/`** (или перетаскивает в чат).
4. **Claude**: смотрит файл (Read) → бракует или принимает → оптимизирует sharp'ом
   (WebP, нужные размеры) → встраивает (CSS/компонент/Media) → PR → деплой → смоук.
   Принятые оригиналы переезжают в `design/accepted/`, чтобы `incoming/` был пустым.

## Правила для генерации (владельцу)

- **Размер**: выбирать максимальное разрешение, какое даёт сервис (минимум 1600 px
  по длинной стороне; для фонов героя — лучше 2000+).
- **Пропорции**: указаны в каждой карточке. В Midjourney — суффикс `--ar 16:9`,
  в Ideogram/других — кнопка выбора aspect ratio. Если сервис не умеет нужную
  пропорцию — генерить ближайшую более «широкую», обрежу сам.
- **Без текста на картинке**: нейросети пишут кириллицу с ошибками. Все надписи
  делаем кодом поверх. Если в результате вылез текст/буквы — перегенерить.
- **Без узнаваемых лиц** крупным планом: люди — со спины, в толпе, силуэтами
  (избегаем и юридических вопросов, и «пластикового» AI-лица).
- Сохранять PNG или JPG максимального качества — сжимаю на своей стороне сам.

## Единый стиль (хвост каждого промта)

Чтобы картинки из разных сервисов выглядели одной серией, в конец КАЖДОГО промта
добавляется один и тот же стилевой блок:

```
festive summer atmosphere, warm sunlight, rich emerald green and crimson red and
golden yellow color palette, Tatar folk festival aesthetic, tulip ornament motifs,
high detail, professional photography style, no text, no watermarks
```

(Для иллюстративных, не фотореалистичных задач блок меняется на
`flat folk art illustration style, ...` — указано в карточке.)

поверх — сцены не выбрасываем.

## Технические заметки (для Claude)

- Оптимизация: `sharp` → WebP q≈80 + fallback JPG; для фонов нарезать 2 размера
  (1920 и 960) под `image-set()`/`srcset`. Вес фона — не более ~250 КБ.
- Куда класть: статичные фоны оформления — `web/public/decor/` (в репо, версионируются
  с кодом — это НЕ контент-медиа); контентные фото (галерея) — через Payload Media.
- VPS маленький (диск 35%) — следить за суммарным весом, исходники в репо не коммитить
  (только оптимизированные); оригиналы остаются в `design/accepted/` локально,
  каталог в `.gitignore`.
- Каждое внедрение — обычный PR-flow + деплой + смоук.
